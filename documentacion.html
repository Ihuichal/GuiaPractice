Semana 2

En este periodo ya había una idea más clara sobre que se tenía que desarrollar, sin embargo no había claridad acerca del producto final.
Se realizó la instalación de los entornos de desarrollo Android Studio y Xcode, luego de ello se efectuaron una serie de tutoriales como punto de partida al desarrollo.
Para el desarrollo de los tutoriales y la familiarización con el desarrollo de aplicaciones móviles, se ocupó la documentación que pone a disposición tanto Android como Apple.
Las pequeñas aplicaciones creadas como pruebas, fueron almacenadas en el repositorio Github y para esto se utilizaron una serie de comandos en la terminal para registrar los avances.
Entre los comandos usados tenemos:
→ git init : para iniciar repositorio en un directorio.
→ git add. : para agregar todo el contenido modificado o nuevo.
→ git commit -m “mensaje” : se utiliza para consignar un conjunto de cambios.
→ git push : usado para subir los archivos.
→ git branch : se utilizó para crear funcionalidades sin dañar la rama principal.
→ git checkout : se usa para moverse entre ramas o commits.
→ git merge : utilizado para fusionar las nuevas funcionalidades creadas en las ramas con la rama master.
